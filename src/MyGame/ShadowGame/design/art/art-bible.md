## Visual Identity Summary

影子回忆的视觉语言建立在"光与影的对话"之上：半写实的 3D 生活场景中，玩家摆放日常物件、调整光源，在墙面或地面上投射出两个人共处的影子轮廓。整体画面克制、温柔、安静——色彩饱和度低，光线柔和但方向感明确，空间中充满生活痕迹却不喧闹。视觉设计的最高优先级是**影子轮廓的可读性**：一切材质、光照、色彩决策都必须服务于"玩家能清晰辨认投影墙上的影子形状"这一核心约束。

## Art Style

### Rendering Style

**3D 半写实（Soft Realism）**，使用 Unity URP 管线。

核心特征：
- **几何写实、质感柔化**：物件造型遵循真实比例和结构，但表面材质略微简化——减少高频细节（如木纹的每一条纹理），保留中频信息（整体材质感受），类似"轻度涂抹后的照片"
- **光影高度写实**：光的传播、衰减、阴影的软硬度遵循物理规则，这是游戏核心机制的基础，不能风格化
- **色彩控制**：整体饱和度偏低（HSB 中 S 值通常在 15-35% 之间），避免纯黑纯白，用带色倾向的深灰和暖白替代
- **边缘处理**：物件边缘微微柔化（细微的抗锯齿 + 极轻度 Bloom），不使用描边/卡通轮廓线

### Proportions

- **物件**：严格 1:1 真实比例。台灯高 40-50cm、杯子高 10-12cm、椅子座面高 45cm——物件在游戏中的相对大小必须符合日常直觉，否则影子轮廓会产生违和感
- **场景空间**：单个关卡的可交互区域约 3m × 3m × 2.5m（桌面/房间一角），相机可观察范围略大于此
- **投影墙面**：始终占据画面 40-60% 的面积，是视觉焦点

### Level of Detail

| 层级 | 精度 | 说明 |
| ---- | ---- | ---- |
| **核心物件**（可摆放/可交互） | 高 | 轮廓清晰锐利，剪影在任何角度都可辨识；面数 500-2000 tris；至少 1 套 PBR 贴图（Albedo + Normal + Mask） |
| **场景物件**（不可交互装饰） | 中 | 提供生活氛围但不抢夺注意力；面数 200-800 tris；可用更简单的材质 |
| **投影墙面** | 低几何/高表面 | 几何极简（平面或微曲面），但表面材质需精心调校——墙面纹理不能干扰影子可读性 |
| **背景/画面边缘** | 低 | 适度虚化或暗化，将注意力导向中心场景 |

### Visual Hierarchy（视觉层次）

玩家视线引导的优先级，从高到低：

1. **影子投影区域**（墙面/地面上的影子）— 全画面最高对比度区域，影子与墙面之间的明暗差必须保持 ≥ 40% 亮度差
2. **当前操作中的物件** — 通过微弱的边缘高光或轻度发光提示可交互状态
3. **光源** — 发光体自身带有柔和光晕，明确指示光线方向
4. **其他可交互物件** — 通过放置区域的微弱标识暗示
5. **场景环境** — 氛围衬托，不争夺注意力

> **铁律**：任何时候，如果场景中的某个视觉元素比影子更抢眼，那就是 bug。

## Character Art Standards

本游戏没有直接可见的角色模型。"角色"仅以影子剪影的形式存在于投影墙面上。

### 影子剪影设计规范

**两个人的影子**是本作的核心"角色"，其设计标准如下：

#### 基础剪影
- **人物 A（"我"）**：中等身材轮廓，肩线较平直，站姿略微内收——传递一种内向、安静的气质
- **人物 B（伴侣）**：身材轮廓与 A 形成差异（略高或略矮，肩线弧度不同），通过轮廓差异实现辨识，不依赖性别标签
- **关键要求**：A 与 B 的剪影在重叠、并列、分离时都必须可区分——通过头部轮廓、肩线、姿态差异实现

#### 影子的 5 章状态变化

| 章节 | 影子状态 | 视觉特征 |
| ---- | ------- | ------- |
| Ch.1 靠近 | 两个独立剪影逐渐接近 | 轮廓清晰硬朗、边缘锐利，像初次见面的"陌生人"形状 |
| Ch.2 共同空间 | 两个剪影开始有互动姿态 | 轮廓依然清晰，但开始出现两影交叠的区域 |
| Ch.3 共同生活 | 两个剪影融为默契的组合 | 单看某些影子，分不清是一个人还是两个人的叠影——这就是"关系" |
| Ch.4 松动 | 影子组合出现错位、断裂 | 本应对齐的轮廓出现偏移；边缘开始出现毛刺/虚化 |
| Ch.5 缺席 | 只有一个影子，或影子不完整 | 缺失的那部分用微弱的光晕/色差暗示"曾经在这里"；影子边缘柔化模糊 |

#### 影子清晰度技术标准
- 硬阴影用于核心谜题影子（Shadow Map 分辨率 ≥ 2048，移动端可降至 1024 但需测试可读性）
- 影子与墙面的明暗比 ≥ 3:1（推荐 5:1）
- 影子边缘的 Penumbra 宽度按章节调节：Ch.1-3 为 0-2px，Ch.4 为 2-4px，Ch.5 为 4-8px
- 投影墙面禁止使用高频纹理（如砖墙缝隙、碎花壁纸），推荐纯色/低频渐变/极微弱噪点

## Environment Art Standards

### 场景构成原则

每个关卡是一个"生活角落"——不是完整的房间，而是一张桌子、一个窗台、一个门口、一面墙前的空间。像是记忆中只保留了最重要的那个局部。

### 空间层级

| 层级 | 内容 | 材质标准 | 说明 |
| ---- | ---- | ------- | ---- |
| **前景层**（z=0-1m） | 可交互物件、光源 | 完整 PBR | 玩家操作的核心区域 |
| **中景层**（z=1-3m） | 投影墙面、桌面、地面 | 简化 PBR，重点在表面光影响应 | 影子投射的画布 |
| **背景层**（z=3m+） | 房间轮廓暗示、窗外光线 | 低面数 + 简单 Unlit/低成本 Shader | 营造空间感但不抢焦 |

### 场景模块化

- 墙面模块：2m × 2.5m 标准墙面 Tile，提供 4-5 种墙面材质变体（刷漆墙/旧木板/水泥/壁纸低频/磨砂玻璃）
- 地面模块：2m × 2m 标准地砖，3-4 种变体（木地板/地砖/地毯/水泥）
- 家具模块：桌、椅、柜、架等以独立 Prefab 存在，可在不同关卡重新组合

### 光照设计

光照是本作的玩法核心，也是美术的最高优先事项。

#### 光源类型

| 光源 | Unity 类型 | 用途 | 视觉特征 |
| ---- | --------- | ---- | ------- |
| **台灯** | Point Light | 最常用，可摆放/旋转 | 暖色球形衰减，较硬阴影 |
| **窗外光** | Directional Light | 固定方向光，提供基础影子 | 冷白色平行光，自然硬影 |
| **蜡烛/小夜灯** | Point Light (低强度) | 营造氛围 | 暖橙色，衰减快，影子柔和 |
| **手电筒/聚光灯** | Spot Light | 特殊关卡机制 | 锥形光束，可旋转改变投影方向 |
| **环境补光** | 间接光/Ambient | 防止非影子区域全黑 | 极低强度冷蓝，不产生独立阴影 |

#### 光照约束
- 同一场景中玩家可控光源不超过 3 个（保证谜题可解且 draw calls 可控）
- 每个光源必须产生清晰的实时阴影——这是谜题成立的前提
- 环境补光强度不超过主光源的 15%，防止"冲淡"影子
- URP Shadow Distance 根据场景大小调整，通常 5-10m 足够
- 所有可交互光源的色温必须与当前章节色彩弧线一致

### 5 章环境氛围

| 章节 | 场景关键词 | 光照特征 | 环境细节 |
| ---- | --------- | ------- | ------- |
| Ch.1 靠近 | 咖啡桌、公共空间、窗边 | 自然光为主，柔和偏白 | 空间整洁但略显空旷，物件少且有距离感 |
| Ch.2 共同空间 | 客厅、书桌、沙发旁 | 暖色台灯出现，自然光+人工光并存 | 开始出现"两个人"的痕迹——两只杯子、两把椅子 |
| Ch.3 共同生活 | 厨房、卧室、阳台 | 最暖的光、最饱满的色彩 | 物件最丰富，空间最"满"，有生活的层叠感 |
| Ch.4 松动 | 同样的空间，但有微妙变化 | 光线开始变冷，出现刺眼的明暗对比 | 某些物件移位了，桌上只剩一只杯子，灯偏向一边 |
| Ch.5 缺席 | 清空中的房间、箱子、门口 | 冷光为主，偶尔闪回暖光 | 物件在消失，空间在变空；最终关卡只剩极少物件和一束光 |

## UI Art Standards

### 设计原则

UI 必须**隐于体验**——玩家的注意力应始终在场景和影子上，UI 只在需要时浮现，不需要时消融于画面。

### 风格定义

- **字体**：无衬线体，笔画纤细（推荐思源黑体 Light/Regular 或类似字体），中英文统一风格
- **颜色**：UI 文字使用 `#F5F0E8`（柔象牙）配合 60-80% 不透明度，在深色场景上保证可读性；按钮/边框使用 `#D4A574`（暖琥珀）的 30-50% 透明度
- **形状语言**：圆角矩形（圆角半径 8-12px @1080p），不用锐角——锐角传递紧张感，不符合本作气质
- **动效**：所有 UI 出现/消失使用 Fade + 微位移（300-500ms，EaseInOut），不用弹性/夸张动画
- **图标**：线性图标（1.5-2px 描边），与字体笔画粗细统一

### 具体元素规范

| 元素 | 规格 | 说明 |
| ---- | ---- | ---- |
| 暂停菜单 | 居中半透明面板，背景 Gaussian Blur | 模糊场景但不完全遮挡，保留光影氛围 |
| 章节标题 | 画面中央，大号字，Fade In/Out | 显示 2-3 秒后自动消失，不设关闭按钮 |
| 操作提示 | 画面底部，小号字，低不透明度 | 首次出现新操作时显示，重复操作不再提示 |
| 物件拾取/放置反馈 | 极轻微的脉冲缩放 + 音效 | 不使用粒子特效或高亮弹窗 |
| 设置界面 | 左侧列表 + 右侧内容，风格与暂停菜单一致 | 音量滑杆、画质选项、辅助功能开关 |

### HUD 策略

**无常驻 HUD**。没有计分板、计时器、生命值。屏幕上唯一持续存在的是场景本身。需要传递的信息通过场景内元素（光线变化、物件状态）和极简的临时提示完成。

## VFX Standards

### 设计原则

特效服务于**光影的真实感和情绪**，不是华丽表演。本作的"特效"更接近自然现象而非游戏特效。

### 特效清单

| 效果 | 实现方式 | 用途 | 性能要求 |
| ---- | ------- | ---- | ------- |
| **灰尘粒子** | GPU Particle（极少量） | 光束中可见的浮尘，增加空间真实感 | ≤50 粒子/场景 |
| **影子成型反馈** | Shader 动画 | 当玩家拼出正确影子时，影子边缘微微发光后锐化 | 单个 Full-screen Pass |
| **光源开关** | Intensity 动画 + Bloom 脉冲 | 台灯打开/关闭时的视觉反馈 | Light Intensity Tween |
| **记忆闪回** | Post-processing Volume 渐变 | 色温瞬间变暖 → 回到当前冷色 | PP Volume Lerp |
| **窗帘飘动** | Shader（顶点位移 Sine Wave） | 窗边场景的环境动态 | 单 Shader，无粒子 |
| **边缘虚化（Ch.4-5）** | Post-processing Vignette + 色差 | 影子不完整时画面边缘的焦虑感/失焦感 | PP Pass |

### 禁止使用的特效

- 屏幕抖动（Screen Shake）
- 粒子爆发/烟花/火花
- 高饱和度闪光
- 运动模糊（会降低影子可读性）
- 过度 Bloom（会模糊影子边缘）

> **总原则**：如果一个特效会降低影子轮廓的可读性，无论多好看都不采用。

## 物件设计规范（Object Design Standards）

物件是本作的核心游戏元素，其设计标准独立于一般环境美术。

### 设计优先级

1. **剪影可辨识**：物件在任何旋转角度下，其投影剪影都必须可被辨识为该物件
2. **日常可信**：物件必须是普通生活中存在的，不用奇幻/sci-fi/抽象物件
3. **组合可叙事**：单个物件是中性的，两个物件的组合暗示一段关系

### 核心物件库

| 物件 | 剪影关键特征 | 叙事潜力 | 章节分布 |
| ---- | ----------- | ------- | ------- |
| 台灯 | 灯罩锥形轮廓 + 支架 | 光源本身也是物件——它照亮其他物件的同时也投射自己 | 全章节 |
| 杯子（×2） | 圆柱 + 把手弧线 | 一只杯子是独处，两只杯子是共处 | Ch.1-5（数量变化叙事） |
| 椅子（×2） | 靠背直线/弧线 + 四腿 | 两把椅子面对面/背对背/只剩一把 | Ch.2-5 |
| 雨伞 | 弧形伞面 + 伞柄直线 | 一把伞下两个人/伞收起立在角落 | Ch.1, Ch.4 |
| 窗帘 | 垂直褶皱轮廓 | 拉开/合上改变光线入射——控制"看见"的程度 | Ch.2-5 |
| 桌子 | 水平面 + 腿 | 承载物件的平台，桌面物件组合讲述日常 | 全章节 |
| 鞋架 | 横档 + 鞋的轮廓 | 两双鞋/一双鞋/空架子 | Ch.2-5 |
| 衣架 | 三角形肩部轮廓 + 钩 | 挂着两件外套/一件外套/空衣架 | Ch.3-5 |
| 钥匙 | 小而锐利的特征轮廓 | 两把钥匙在一起/分开/只剩一把 | Ch.2-5 |
| 书籍/相框 | 矩形薄体 | 翻开/合上/面朝下——暗示"是否愿意回看" | Ch.3-5 |

### 物件建模规范

- 面数预算：500-2000 tris/物件（核心可交互物件）
- 必须提供**明确的轮廓边（Hard Edge）**——影子由轮廓决定，内部细节在投影中不可见
- 物件底部需要平整的接触面，用于放置交互的物理稳定性
- 每个物件必须提供 Mesh Collider 或简化的 Box/Capsule Collider 用于拾取交互
- UV 展开优先保证物件在相机正视角度下的贴图精度

### 物件材质原则

- **不追求 PBR 极致写实**，但需要有"被使用过"的质感：木头有磨痕、杯子有微渍、金属有指纹感
- 材质反光不能过强——高 Specular 会产生高光点干扰影子区域的对比度阅读
- Metallic 值通常 ≤ 0.3，Smoothness 值通常 ≤ 0.5
- 物件颜色倾向低饱和的自然色系：木色、米色、灰白、暗绿、深蓝

## Post-Processing Pipeline

使用 URP Volume System，按章节配置不同的 Post-Processing Profile。

### 全局固定效果

| 效果 | 参数范围 | 说明 |
| ---- | ------- | ---- |
| Tonemapping | ACES | 统一色调映射 |
| Color Adjustments | Saturation: -10 ~ -25 | 全局轻微降饱和 |
| Bloom | Intensity: 0.3-0.8, Threshold: 0.9 | 极轻度 Bloom，仅光源处可见 |
| Vignette | Intensity: 0.2-0.35 | 画面边缘微暗，引导视觉中心 |
| Anti-aliasing | SMAA (Medium) on mobile, SMAA (High) on PC | 保证影子边缘平滑 |

### 章节可变效果

| 效果 | Ch.1 | Ch.2 | Ch.3 | Ch.4 | Ch.5 |
| ---- | ---- | ---- | ---- | ---- | ---- |
| Color Temperature | +5 | +15 | +20 | -10 | -15 → +5 |
| Saturation Offset | -15 | -10 | -5 | -20 | -25 → -15 |
| Vignette Intensity | 0.2 | 0.2 | 0.25 | 0.3 | 0.35 → 0.25 |
| Chromatic Aberration | 0 | 0 | 0 | 0.05 | 0.1 → 0 |
| Film Grain | 0 | 0 | 0.02 | 0.05 | 0.08 → 0.03 |

> Ch.5 的 `→` 表示该章内部从"缺席最冷"过渡到"释然回暖"的参数动画。

## Performance Budget（移动端）

| 指标 | 目标 | 硬上限 |
| ---- | ---- | ------ |
| 帧率 | 60fps | ≥45fps（任何情况不低于此） |
| Draw Calls | ≤100/帧 | ≤150/帧 |
| Tris/帧 | ≤80K | ≤120K |
| 实时阴影光源 | ≤2 | ≤3 |
| 贴图内存/关卡 | ≤30MB | ≤45MB |
| 总内存 | ≤1.2GB | ≤1.5GB |
| Shader 变体 | ≤50 | ≤80 |
| 粒子数/场景 | ≤50 | ≤100 |

### 性能优化策略

- **SRP Batcher** 启用，所有材质使用 URP/Lit 或自定义 SRP-compatible Shader
- **Static Batching** 用于所有不可移动的环境物件
- **GPU Instancing** 用于同类物件多实例（如多把椅子）
- **Shadow Cascade** 移动端使用 2 级 Cascade，PC 使用 4 级
- **Texture Streaming** 启用，按需加载非当前关卡资源
- **物件池** 用于跨关卡重复使用的物件 Prefab

## Accessibility（无障碍设计）

### 色觉辅助
- 影子可读性不依赖色相区分——影子与墙面的区别完全基于明度差异，天生色觉友好
- UI 中所有功能性色彩提示必须同时使用**图标 + 颜色**，不单独依赖颜色
- 提供"高对比度影子"选项：开启后影子颜色加深至纯黑 `#000000`，墙面亮度提升

### 视觉辅助
- 最小文字尺寸：28px @1080p（移动端手持距离下的可读性保障）
- 提供"影子描边辅助"选项：开启后在影子边缘添加 1-2px 亮色描边，极大提升轮廓辨识度
- 所有 UI 文字支持 3 档字号切换：标准 / 大 / 超大
- 按钮/可交互区域最小触控尺寸：44×44pt（Apple HIG 标准）

### 操作辅助
- 物件拖拽支持"吸附到网格"选项，降低精确摆放的操作难度
- 光源旋转支持"步进角度"选项（如每 15° 一档），降低微调难度
- 提供"影子提示"功能：长按影子区域，用虚线提示正确轮廓的大致方向

### 听觉辅助
- 所有叙事性音频提供字幕
- 关键音效反馈同时伴随视觉反馈（如物件放置成功的轻微脉冲缩放）

## Technical Art Notes

### Shadow 实现建议（供 Tech Art / 程序参考）

- 使用 URP 的 Additional Light Shadows 实现多光源实时阴影
- 核心谜题影子建议使用独立的 **Projector / Decal** 系统实现精确控制，不完全依赖 URP 默认 Shadow Map
- 投影墙面使用专用 Shader：接收阴影但自身材质极简，可在 Fragment Shader 中增强影子的对比度
- 影子"成型判定"使用 Render Texture 采样：将投影墙面区域渲染到 RT，与目标剪影做像素比对
- 考虑 **Shadow Only** 模式的不可见物件——某些谜题可能需要"看不见物件本体，只看到它的影子"

### 材质工作流

- 基础材质：URP/Lit（Metallic Workflow）
- 投影墙面专用材质：自定义 Shader Graph，确保阴影接收 + 可控对比度
- 物件材质变体通过 Material Property Block 控制，避免材质实例化爆炸
- 所有材质必须在 URP Shadow 开启状态下测试视觉效果，不接受"关掉阴影更好看"的材质
